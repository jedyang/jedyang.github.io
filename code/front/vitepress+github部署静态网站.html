vitepress+github部署静态网站 ​

这里主要记录遇到的问题。使用vitepress网上文章很多。

项目结构 ​

vitepress编译后的静态资源是放在

docs/.vitepress/dist目录下

所以，按理说我们网址应该是发布这个目录。

但是我使用的github action脚本peaceiris/actions-gh-pages@v3

会把这个目录的推送到gh-pages分支上。

所以，我的站点发布的时候就选择gh-pages分支，发布其根目录。

这是目前比较通用的做法。

github仓库问题 ​

  1. 需要是public的，才能使用github pages功能

    所以如果不想自己的源码公开，就需要搞两个仓库。一个私有的放源码，一个公开的放编译后的静态文件。

    也就是我目前的部署方案

  2. 仓库的命名应该是 用户名.github.io

    这也是默认的访问域名。如果你有自己的域名也可以配置。

github action问题 ​

官方仓库 ​

GitHub Marketplace · Actions to improve your workflow

在这里找需要的actions

自动部署 ​

脚本文件的路径是项目的根目录下 .github\workflows\deploy.yml

名字随便起。github会自动在.github\workflows下找yml文件执行

我的github action脚本

name: Deploy

on:
  push:
    branches:
      - main # 表示对main分支的push会触发

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
        with:
          fetch-depth: 0
      - uses: actions/setup-node@v3
        with:
          node-version: 20
          cache: yarn
      - run: yarn install --frozen-lockfile

      - name: Build
        run: yarn docs:build

      - name: publish # 发布到另一个仓库
        uses: s0/git-publish-subdir-action@develop
        env:
          REPO: git@github.com:jedyang/jedyang.github.io.git
          BRANCH: gh-pages
          FOLDER: docs/.vitepress/dist
          SSH_PRIVATE_KEY: ${{ secrets.DEPLOY_PRIVATE_KEY }}

同步到gitee ​

我使用的是这个action工具 Gitee Pages Action · Actions · GitHub Marketplace

在静态文件仓库jedyang.github.io的main分支加一个action脚本

name: syncToGitee
on:
  push:
    branches:
      - gh-pages
jobs:
  repo-sync:
    runs-on: ubuntu-latest
    steps:
      - name: Sync to Gitee
        uses: wearerequired/git-mirror-action@master
        env:
          # 注意在 Settings->Secrets 配置 GITEE_RSA_PRIVATE_KEY
          SSH_PRIVATE_KEY: ${{ secrets.GITEE_RSA_PRIVATE_KEY }}
        with:
          # 注意替换为你的 GitHub 源仓库地址
          source-repo: git@github.com:jedyang/blogs.git
          # 注意替换为你的 Gitee 目标仓库地址
          destination-repo: git@gitee.com:helloyunsheng/blogs.git
